## Context

当前页面为单 HTML 文件（`static/index.html`），使用 MapLibre GL 渲染地图，左侧 380px 固定侧栏包含所有 UI 控件。所有功能共享一个页面状态，无模式概念。提交成功后结果面板在侧栏底部展示，需要滚动才能看到完整结果。

## Goals / Non-Goals

**Goals:**
- 将输入区和查看区在视觉和交互上清晰分离
- 提交成功后自动切换到查看模式，用户无需手动滚动
- 支持在两种模式间自由切换，切换时不丢失数据
- 地图根据模式自适应：输入模式显示引导状态，查看模式聚焦到战役数据
- 仅修改 `static/index.html`，不动后端

**Non-Goals:**
- 不改变任何后端 API
- 不改变数据模型或管道逻辑
- 不改变可编辑面板的内部逻辑（编辑/删除/添加/重新渲染均保留）
- 不支持多标签页/多战役对比

## Decisions

### 1. 模式切换机制：CSS class 控制显隐

使用 body 或根容器上的 CSS class (`mode-input` / `mode-view`) 驱动两个模式的显示/隐藏，而非 JS 动态创建/销毁 DOM。

**理由**：
- 切换模式时所有 DOM 状态（文本框内容、编辑结果）自然保留
- CSS transition 可以实现平滑过渡动画
- 代码量最小，不需要复杂的框架

**方案**：
```css
.input-only { display: flex; }
.view-only { display: none; }
body.mode-view .input-only { display: none; }
body.mode-view .view-only { display: flex; }
```


### 4. 地图三种状态

| 状态 | 触发条件 | 地图行为 |
|------|---------|---------|
| 引导态 | 页面初次加载 / 返回输入模式且无数据 | center [112,33] zoom 5，显示提示覆盖层 |
| 结果态 | 提交成功进入查看模式 | fitBounds 聚焦到数据范围，scale 控制 maxZoom |
| 编辑态 | 用户在查看模式中点了重新渲染 | 同结果态，但不改变模式 |

### 5. 模式切换时的数据保持

- 切换到查看模式：保留文本框内容（用户可能想回去修改）
- 切换到输入模式：保留 `_lastExtractData` 和地图上的 markers
- 编辑面板修改后点「返回编辑」→ 地图保持当前渲染结果，直到下次提交覆盖

## Risks / Trade-offs

- [单文件体积增长] → 当前 index.html 约 1060 行，改版后可能到 ~1300 行。前端简单到不需要拆分 JS 文件，可接受。
- [模式切换动画不流畅] → 使用 CSS `opacity` + `max-height` transition，在移动端可能掉帧。降级方案：去掉动画，直接 display 切换。
- [查看模式下侧栏仍然只有 380px] → 编辑面板的空间瓶颈不变。但这是单页面架构的固有限制，暂不引入侧栏拖拽调整宽度（属于后续优化）。
