## Context

Qwen-VL 视觉审查暴露了三个根因在提取管道和后端渲染的问题。当前 `_TIMELINE_SYSTEM_PROMPT` 缺少明确的阵营识别规则，导致 LLM 在不提供显式阵营名的文本中（如测试文本只提"王贵""张宪"不提"宋军"）无法正确归类。方向字段虽有规则 7 约束，但 LLM 对模糊战术语境（"据守""迂回包抄"）的推断覆盖率不足。行军路线虚线在当前 `[8,4]` dash pattern 下间隙过大，视觉上被误判为断裂。

## Goals / Non-Goals

**Goals:**
- 提高 LLM 对阵营（faction）的识别准确率，确保部队标记颜色与阵营一致
- 提高 direction 字段的提取覆盖率，减少"无方向"部队
- 优化行军路线虚线样式，减少视觉断裂感
- 为路线端点添加圆形锚点标记，使路线与地名的空间关系更清晰

**Non-Goals:**
- 不改变数据模型（Pydantic 字段不变）
- 不引入新的外部依赖
- 不改变 API 接口签名
- 不调整 UI 布局/字号等纯 cosmetic 问题（改动面积大、收益低）

## Decisions

### Decision 1: Prompt 中新增阵营识别规则

**方案**: 在 `_TIMELINE_SYSTEM_PROMPT` 新增规则 10，指导 LLM 从历史语境推断阵营归属。

核心逻辑：
- 优先使用文本中明确提及的阵营名
- 若未明确，根据历史常识推断：宋代背景下"我方"通常为宋，"敌方"通常为金/西夏/辽
- ForceUnit.faction 必须与 factions[].name 精确匹配
- faction 名使用标准历史称谓（"宋""金""西夏""辽""蒙古"等），而非"敌军""我军"等指代词

**替代方案**: 在 `_filter_military_unit_places()` 同级新增后处理过滤器，硬编码将领→阵营映射表。
**不采用原因**: 硬编码映射表无法泛化到所有历史文本，维护成本高且容易过时。

### Decision 2: 强化方向提取规则的示例

**方案**: 在规则 7 中增加正反例说明，帮助 LLM 理解"据守""迂回"等战术动词如何映射到八方位词。

新增示例：
- "据守X地" → 无进攻方向，direction=null ✓
- "从岭北仰攻" → 攻击方向向南，direction="南" ✓  
- "自岭南迂回包抄" → 向南包抄，direction="南" ✓
- "向塬底冲击" → 向塬底方向，需判断塬底相对位置 → direction="西北" ✓
- "侧翼压上" → 侧面推进，"侧翼"不是方位词 ✗

**替代方案**: 在后端 `_make_unit_geojson()` 中根据 unit_state.location 坐标相对上一个位置的偏移自动计算方向角。
**不采用原因**: 与 spec 设计冲突——spec 规定 direction 由 LLM 提供语义方向，compute 方式丢失了战术含义（"向南进攻"和"位于南方"不是同一含义）。

### Decision 3: 路线虚线 dash pattern 收紧

**方案**: 将 `route-lines` 默认 `line-dasharray` 从 `[8, 4]` 改为 `[6, 3]`，comic 主题从 `[10, 5]` 改为 `[6, 3]`。间隙从 4px 缩至 3px，视觉上更连续。

**替代方案**: 改为实线。不采用——虚线是古地图视觉体系的一部分（spec 定义），去掉会失去风格一致性。

### Decision 4: 前端生成路线锚点标记

**方案**: 在 `updateMap()` 中从每条 route LineString 提取首末坐标，构造成 Point GeoJSON，通过独立的 `route-anchors` source 和 circle layer 渲染为小圆点。样式：直径 6px，朱砂色 `#c23b22`，opacity 0.6，无交互。

这比在 `app.py` 后端 `_make_geojson()` 中生成锚点更简单，不需要修改后端代码和 SSE 管道。

## Risks / Trade-offs

- **Prompt 变更可能影响已有提取质量**: LLM 对 prompt 措辞敏感，新增规则可能意外改变其他字段的行为。→ 通过现有测试套件回归验证，必要时调整 prompt 措辞。
- **方向提取覆盖率提升有限**: LLM 推断方向本质上是语义理解任务，prompt 改进只能边际提升，无法保证 100% 覆盖。→ 接受合理范围（80%+），无方向时前端正常渲染旗帜（仅无方向线）。
- **锚点标记可能增加视觉噪音**: 在密集战役图上，路线的首末点圆圈可能与地名/部队标记重叠。→ 使用小尺寸 + 低透明度 + 无交互，保持视觉低调。
