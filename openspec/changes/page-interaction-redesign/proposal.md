## Why

当前页面将所有功能堆叠在 380px 侧栏中——截图上传、文本输入、提交按钮、进度条、结果面板、时间轴、事件卡片全部挤在一起。用户需要频繁上下滚动才能完成「输入→查看结果→编辑→重新渲染」的完整流程。输入区和结果区混排导致视觉混乱、操作路径长，且地图在战术级文本时大面积空白浪费空间。将输入和查看分离为两个独立模式，可以显著改善操作流畅度和空间利用率。

## What Changes

- 新增**模式切换机制**：输入模式（编辑文本/上传截图）和查看模式（结果面板/时间轴/地图交互）相互独立
- 输入模式下侧栏仅保留截图上传区、文本框、朝代选择、提交按钮，地图占满右侧
- 查看模式下侧栏显示结果面板（可编辑地名/路线/阵营）、时间轴控件、事件卡片，输入区折叠为顶部摘要条
- 提交成功后自动切换到查看模式，点击「返回编辑」切回输入模式
- 地图在输入模式下显示为引导状态（中国全景 + 提示文字），查看模式下聚焦到战役位置
- 结果面板中已有的可编辑功能和重新渲染按钮保持不变

## Capabilities

### New Capabilities
- `page-mode-switching`: 输入模式与查看模式之间的切换逻辑、动画过渡、状态保持

### Modified Capabilities
- `campaign-map-rendering`: 地图在输入/查看两种模式下采用不同的初始状态和交互行为
- `timeline-interaction-ui`: 时间轴控件从侧栏底部移至查看模式专属区域，与结果面板整合

## Impact

- `static/index.html`：单文件全部改动——CSS 布局重构 + HTML DOM 结构调整 + JS 模式切换逻辑
- 不影响任何后端 API（app.py、extractor、ocr 等均无变动）
- 不影响现有数据模型和管道逻辑
